# NLUX Marketing Plan — Initial Outline

**Version:** 0.1.0-draft  
**Date:** 2026-04-03  
**Status:** Draft

---

## 1. Positioning

**NLUX** is the open, linked-data collections discovery platform for Dutch cultural heritage.  
Built on world-class technology (Yale LUX), adapted for Dutch institutions, with no vendor lock-in.

**Tagline (working):** *"Connected collections. Open by design."*

---

## 2. Target Audiences

| Audience | Their Need | What NLUX Offers |
|----------|-----------|-----------------|
| Collection managers (Teylers, Boerhaave, etc.) | Better findability, less IT burden | Managed open platform, easy onboarding |
| IT / digital teams at institutions | Interoperability, standards compliance | Linked.art, open APIs, LUX compatibility |
| Grant officers / directors | Align with NDE / Europeana strategy | DERA/NDE alignment, clear impact story |
| Researchers / Digital Humanists | Machine-readable, linked collections data | Open APIs, SPARQL endpoint, Linked.art data |
| General public | Discover Dutch heritage online | Beautiful, fast, multilingual discovery UI |

---

## 3. Key Messages

1. **Open and free**: No licence fees. Code is Apache 2.0. Your data stays yours.
2. **World-class technology**: Built on Yale LUX — proven at scale in one of the world's great universities.
3. **Linked data without the pain**: Full Linked.art / CIDOC-CRM compliance, with Dutch authority files integrated out of the box.
4. **Small team, big impact**: Affordable for medium and small institutions that cannot build this themselves.
5. **Community, not a vendor**: A foundation, not a company. Governed by its users.

## 4. Channels

### 4.1 Community and Sector Networks
- NDE (Netwerk Digitaal Erfgoed) events and working groups
- DEN (Digitaal Erfgoed Nederland) community forums
- FARO (Flemish heritage network) — future cross-border expansion
- Linked.art community calls and GitHub discussions
- Museums Vereniging and sector associations

### 4.2 Conferences and Events
- Museums & Web (international)
- DH Benelux
- CIDOC annual conference
- Europeana Tech
- Open Science NL

### 4.3 Digital Presence
- Project website (GitHub Pages initially): `nlux-project.org` or `nlux.nl`
- GitHub organisation: `nlux-project`
- Mastodon / LinkedIn for project updates (no Twitter/X)
- Monthly newsletter (Buttondown or similar, free tier)

### 4.4 Publications and Outreach
- Blog posts on implementation experiences, Linked.art tips
- Case studies per pilot institution
- Technical papers for DH / GLAM conferences

---

## 5. Launch Strategy

### Phase 0 — Stealth / Build (Months 1–6)
- Engage pilot institutions in private conversations.
- Build proof-of-concept with Teylers data.
- Prepare website and GitHub organisation.

### Phase 1 — Soft Launch (Month 6–9)
- Announce at one NDE event or working group.
- Publish GitHub organisation and documentation site.
- Release pilot demo URL.
- Targeted outreach to 5–10 institutions.

### Phase 2 — Public Launch (Month 10–12)
- Press release / blog post on project website.
- Submit talk proposals to Museums & Web and DH Benelux.
- Launch newsletter.
- Report on pilot results.
